数据结构要点

六 03 九月 2016
By yanggang

大中型服务器软件一般应考虑以下几点：

* 数据的安全性
* 结构清晰（尽量减少耦合）
* 易扩展性
* 易使用
* 易维护、易升级（日志信息充分，易定位问题）
* 数据的处理效率
* 平台的可移植性
* 保证性能的前提下，尽量少占用CPU和内存

  1. 手写链表，要注意头节点、尾节点的操作，还有参数的合法性，都是得分点，不可大意。

  2. 队列的原理在实际项目中非常广，栈的应用多在库函数中，实际项目比较少见。

    经典题：如何用两个栈实现队列的功能？

  3. 实现队列有单链表和双链表两种方法。 单向链表实现队列可以从头部增加节点，从尾部删除节点，也可以反过来。但是必然有一个操作要遍历整个列表，效率低。因此可采用双向循环链表，减少查询链表尾节点的时间。代价是内存上需要在每个节点上增加一个指针，一个指针在32位系统上占4个字节，如果最大支持1万个节点，那么将比单向链表多占用40M空间，这点内存对于大型服务器软件来说，完全可以忽略不计。

  4. 一般会采用三种方法来遍历一棵二叉树，前序遍历、中序遍历、后序遍历。 前序遍历：（1）根节点，（2）左子树所有节点，（3）右子树所有节点。（子树同样采用前序遍历）

    中序遍历：（1）左子树所有节点，（2）根节点，（3）右子树所有节点。（子树同样采用中序遍历）

    后序遍历：（1）左子树所有节点，（2）右子树所有节点，（3）根节点。（子树同样采用后序遍历）

  5. 排序二叉树（二叉查找树） 特点：

    （1）左子树节点全部小于根节点的值，右子树节点全部大于根节点的值；

    （2）没有键值相等的节点；

    （3）子树也是排序二叉树。

  6. 平衡二叉树：节点左子树的高度与右子树的高度相差不超过 1 插入新数据可能导致二叉树不平衡，无法得到比链表更高的效率，因此需要平衡算法。当树很大的时候平衡算法很复杂。

  7. 二分查找算法：每次查询某数字都与数组中间的元素进行比较。

    面试：二分查找的递归实现和非递归实现（要熟练）。

  8. 运行查找算法的前提条件：（1）数据必须要有序，（2）数据必须是放在连续的存储空间里面的。

  9. 哈希查找 哈希散列算法：按照某种规律把具有相同特性的数据进行分组，这样就有很多组了，或者说很多列了，而这些列之间是没有任何关系的，所以叫散列。

    比如：把学生按学号除以11的余数不同，将其分为11个组。

    哈希表中存放列表地址的数组称为桶（bucket）。比起链表算法，哈希算法需要多用一些存储空间，这个存储空间就是桶的存储空间。

  10. 静态查找表、动态查找表

  11. 异或操作交换两个变量的数据：

    a[i] = a[i] ^ a[i+1];
a[i+1] = a[i] ^ a[i+1];
a[i] = a[i] ^ a[i+1];

    但要注意，如果两个变量值相等则不能使用此方法，因为异或会得到0.

    当然，交换变量值也可以用以下代码实现：

    a = a + b;
b = a - b;
a = a - b;

但是，这种做法有个致命缺陷，就是越界问题。软件工程师在使用加减乘除时要时刻注意是否存在越界问题。

  1. 冒泡排序、选择排序、快速排序

  2. 一般来说，二分算法在笔试里经常出现，而哈希算法在第二轮面试里经常出现。二分算法查找效率很高 log2(n)，但必须保证整个数据在内存空间是连续的，当删除一个数据时，需要把后面的数据都往前挪一个位置，当数据多的时候比较消耗CPU。所以，二分算法适合那些不需要经常增删改的数据，比如全国行政区划列表。而哈希算法适合那些经常增删改的数据，对哈希表操作类似于对链表进行操作，虽然其效率比二分差，但总体来说还是不错的。

  3. 字节对齐 基本是面试必考题。

    typedef struct employee{
    char sex;
    int length;
    char name[10];
} em_st;

请问sizeof(em_st)的值是多少？很多人会回答15，其实并不是。 在定义的数据结构中，如果出现了三种基本的数据类型 char, short int , int ,我们就以占用内存最大的那个作为字节对齐的方式。比如只有char、short int时，就以2字节对齐。只有char，就以1字节对齐。

  1. 结构成员变量中的位 有时候为了节省空间，结构体中的成员变量是按位操作的，比如

    typedef struct test{
    char a:5;   // 第一字节的前5位
    char b:3;   // 第一字节的后3位
    char c:8;   // 第二字节的前8位
    char d:2;   // 第三字节的前2位
    char e:6;
    char f:7;
    char g:1;
} test_em;

    IP地址就可以采用这种形式来定义，比字符串更省空间。

  2. 更改系统默认的字节对齐 代码前加上 pragma pack(4) 就表示采用4字节对齐。当然，实际情况与系统、CPU、编译器版本都有关。一般记住默认以4字节对齐即可。

  3. container_of宏 内核最经典的宏。

    作用：从结构体成员变量的指针获取整个结构体的指针。
