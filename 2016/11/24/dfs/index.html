分布式文件系统

Nov 24, 2016 | technology | Hits

分布式文件系统（Distributed File System）是指文件系统管理的物理存储资源不一定直接连接在本地节点上，而是通过计算机网络与节点相连。分布式文件系统的设计基于客户机/服务器模式。一个典型的网络可能包括多个供多用户访问的服务器。另外，对等特性允许一些系统扮演客户机和服务器的双重角色。

  1. 背景
    为解决系统存储问题，目前使用较多的是NFS(Network File System)。NFS使得客户端可以像操作本地硬盘一样操作另一台服务器上的文件。但NFS有以下缺点：

    • 文件始终都只存储在一台机器上。存储的容量不可能突破物理服务器的上限。
    • NFS并不考虑failover、backup等高可靠性（HA）功能。
    • 大并发时磁盘吞吐量较大，容易带来性能的瓶颈。
      为此，很多公司提出了分布式文件系统的概念，典型的如google的GFS，hadoop的HDFS，memcached公司的MogileFS等。与NFS不同，GFS和HDFS面向的应用一般是单次写入多次读取，且写入一般都是“增加”操作，“修改”操作很少。这种DFS适用于数据集中采集后大量使用的场景，比如搜索引擎、网络分析、海量数据存储等系统。
      DFS应该可以运行在大规模、不同的廉价器件上，并且可以动态扩展。为增加系统可靠性，可将服务器分成多个cluster，部署在不同地域。由于使用的是廉价器件，所以单个硬件系统的故障不再是异常情况，而应当作常见情况进行处理。
      在存储方面，为了提高IO性能和索引效率，数据按照数据块的格式存储，每个数据块缺省设置为64M。

  2. 结构
    整个DFS基本由三部分组成：Master Server、Data Server和Client。其中Master Server负责提供整个文件系统的目录信息，并管理所有cluster中的Data Server。Data Server负责存储文件分块。Client负责提供访问DFS的能力。
